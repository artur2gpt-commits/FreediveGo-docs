## Goal

Move FreediveGo toward a stable Closed Beta launch.

---

## Rules

1. Stability before new features.

2. Bug fixes before expansion.

3. Reuse existing code whenever possible.

4. Do not create duplicate systems.

5. Preserve existing functionality.

6. Minimize regressions.

7. Keep UX simple.

8. Follow approved project decisions.

---

## Required Reviews

Before implementation:

- Product Manager
- Solution Architect

After implementation:

- QA Engineer
- UX Reviewer
- Security Engineer

Before release:

- Release Manager

## Forbidden

Do not:

- Rewrite working systems
- Introduce unnecessary complexity
- Create duplicate UI flows
- Break existing messaging
- Break authentication

## Success Criteria

The platform becomes more stable, more maintainable and closer to Closed Beta launch.
